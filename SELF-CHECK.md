# SELF-CHECK.md — 安装验收自检单

> 读者:装完 [setup.md](setup.md) 全部步骤、准备写交付报告的安装 AI。逐项过,不合格**当场修再检**,全绿才进交付。
> 本单只验「装没装对」;「治理好不好」由心跳与事故管道长期回答,不在此打分。
> 每项都要可证伪证据(命令+真实输出/文件链接),与宪法宣称纪律同标准——自检报告里写「已检查」而拿不出证据,本身就违反你刚装的红线。

## 逐步验收(与 setup.md 一一对应)

### 第 0 步 · 环境探测
- [ ] 三问答案(运行环境 / 全新或存量 / 有无 CI)已记录进交付报告;
- [ ] 所选载体与 [ADAPTERS.md](ADAPTERS.md) 对应列一致;降级承载的机制在 registry 里写的是**实际**级别(抽一条核对:如无会话 hook 的环境,收尾校验载体写的是 L1/CI 而非 hook)。

### 1.1 · 结构层
- [ ] CI 在一个空提交上**真实跑过一次**(附运行链接或输出摘要);
- [ ] pre-commit 已装(`git config core.hooksPath` 输出 `.githooks`)且样例中无残留 `{占位命令}`——没有对应工具的段落已删除,不是留着空转。

### 1.2 · 宪法
- [ ] `wc -l` ≤ 150(附真实数字);
- [ ] 九段一段不缺:意图纲领 / 不变量 / 红线 / 两条决策规则 / 任务契约 / 新原语对齐门 / 宣称纪律 / boot+landing / 知识沉淀+指针;
- [ ] 两条实测红线原文在(不假全绿 / 不虚报完成);
- [ ] 意图纲领草稿已列入交付报告第 ① 项待负责人确认(或如实标注「待口述」);
- [ ] 全文搜 `{` 无残留占位符。

### 1.3 · governance/ 五件套
- [ ] registry 条数 ≤30,每条六字段齐全(ID/摘要/载体/层/出处/死亡条件+复审);
- [ ] 代际登记行已填真实模型标识 + 日期(不是占位符);
- [ ] 宪法里每条红线/协议都能在 registry 找到户口(逐条对照过);
- [ ] incidents / questions / cases 三件表头规则完整(棘轮必填 / 回答强制入库 / 判例四字段),示例占位行已删或已被真实条目替代。

### 1.4 · 状态层
- [ ] ROADMAP(游标 + 约束登记段 + 战线表)、CHANGELOG、docs/index.md 三件存在;
- [ ] ROADMAP 有真实的第一条战线与游标(不是占位符);docs/index.md 已登记项目现有文档资产。

### 1.5 · 心跳
- [ ] 本地 `node scripts/weekly-governance-review.mjs` 跑通,输出完整对账包(附输出摘要);
- [ ] `GOV_REVIEW_DATE={下月第一个周一}` 复跑,「月度腐烂审计」段出现(月度分支验证);
- [ ] workflow 已挂上(或非 GitHub 环境的等效定时器已配置),且该心跳已在 registry 上户口;
- [ ] AI 判断层状态写进交付报告:已配 key 启用 / 未配(附「配 key 即启用」说明)。

### 第 2 步 · 首任务演练
- [ ] CHANGELOG 出现演练的流水行(真实任务,走完契约三行 → 执行 → 附证据宣称 → landing 五步 → 提交);
- [ ] 演练中走不通而当场修改的条款,修改本身也记入了 CHANGELOG(没有任何修改也如实说明——首跑零摩擦更该起疑)。

### 第 3 步 · 交付报告
- [ ] 五项齐全:① 待确认清单 ② 文件清单 + registry 全表 ③ 没装什么+为什么(「刻意不装清单」原样在)④ 可选启用项 ⑤ 演练记录。

## 反模式自查(三问,答案都该是「没有」)

- [ ] **没装编码规范/风格指南吧?** 行为实验实证:通用过程规范裸跑组也全数自发达标——装了是白付的注意力税。第一次真实风格分歧时再立,且要有那次分歧作出处。
- [ ] **没建审批流吧?** 验证分层从「全部低风险」起步,由事故校准;day-1 审批流是没交过学费的官僚,只会教会执行者绕路。
- [ ] **没写预防式规则吧?** 抽查 registry:除 day-1 底座与两条实测红线外,每条规则的出处都答得出「哪次真实失败证明它值得留」;答不出的删掉再交付——宁可漏管,不可稀释。
- [ ] 能力清单:docs/capabilities.md 已生成、`--check` 挂 pre-commit+CI、宪法含「先查清单」行(setup 1.6)
