# SETUP — 让 AI 建立项目治理体系（一次性安装 · 给 AI 的命令）

> **用法**：把本文件交给你项目的 AI，说「读 setup.md，为本项目建立治理体系」。
> **AI 执行**：按下面**建立全部文件**，缺哪个建哪个，建完才做其他任务。这是命令，不是参考。
> 本文件是**安装脚本**——建完使命完成（可删或留作下次参考）。**建出来的文件才是常驻治理产物，别和本文件搞混。**

---

## 你要为本项目建立这套体系（结构照此，内容填项目自己的）

### ① `CONSTITUTION.md` — 北极星（≤30 行 · 不变真理）
必须含：
- **使命**：一句话——本项目把「什么乱状态」变成「什么有序状态」。
- **4 条真理**（L1>L2>L3>L4，冲突时高位先）：
  - L1 真相之源唯一：指定唯一 SoT；AI 不伪造、不隐瞒、不猜测，必要时说「不知道」。
  - L2 AI 是合伙人不是助手：挑战、追本质、暴露冲突；不顺从。
  - L3 知识必须沉淀：是 AI 的职责，不是用户的指令。
  - L4 人管不变的、AI 管会变的：人定使命+原理+架构，AI 编译细节+维护。
- **修订机制**：AI 不得自动改本文件，改必须用户拍板。

### ② `AGENTS.md` — AI 工作规则全集（≤80 行 · Codex 也读这个）
必须含 8 节，每条溯源到上面某条真理（标 `[来自 L?]`）：
- §0 身份 · §1 启动协议（开场依次读哪几个文件）· §2 对话中（想法落到哪个文件）· §3 结束前（必跑：沉淀 + lint）· §4 自主治理边界（自主修 vs 等用户拍板）· §5 约束（不可违反的红线）· §6 归档路由（什么内容去哪）· §7 Git 提交规范

### ③ `CLAUDE.md` — Claude Code 启动入口 + 行为铁律
必须含：
- **启动顺序**：CONSTITUTION → AGENTS → 知识索引 → PROJECT。
- **5 条工作铁律**：① 动手前核实（有成本操作前三问：做过没/真状态/权威记录，过不了不动手）② 简洁防过度 ③ 精准修改（只动任务相关）④ 目标驱动（先定成功标准）⑤ 验证后宣称（实证，别拿残缺信息下结论）。

### ④ `PROJECT.md` — 当前坐标（≤30 行）
必须含：当前阶段（一句话）· 当前焦点 · 指针（链向其他治理文件）。

### 项目变大后再建（多人 / 多模块 / 决策频繁 / 跨多会话）
- `ROADMAP.md`（主线表 + 开放问题）· `CHANGELOG.md`（事件流水，每轮追加）· **lint 门禁**（断链 / 行数 / 状态新鲜度——让机器强制，纯自觉必衰减）。

---

## 建完之后（常驻运转，写进上面的 AGENTS.md）

- 每次启动：读 CLAUDE.md → CONSTITUTION → AGENTS → PROJECT。
- 每轮结束：更新 PROJECT 焦点 + CHANGELOG 追加一行。
- 想懂这些规则背后的「为什么」（定律 / 机制 / 踩坑）：读同库 [`governance-architecture-essence.md`](governance-architecture-essence.md)（给人理解，AI 安装只需本文件）。
