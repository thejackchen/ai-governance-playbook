# SETUP — 让 AI 为本项目建立治理体系（一次性安装 · 给 AI 的命令）

> **用法**：把本文件交给你项目的 AI，说「读 setup.md，为本项目建立治理体系」。
> **AI 执行**：按下面**建出全部文件**，缺哪个建哪个，建完才做别的任务。这是命令，不是参考。
> 本文件是**安装脚本**——建完使命完成（可删，或留作下次参考）。**建出来的文件才是常驻治理产物，别和本文件搞混。**

---

## 这套治理的内核（先懂，再建）

不是「一套管 AI 的制度」，而是「给 AI 的一份好交接」。目标：任何 AI（含无记忆、换代、换工具）接手就能 ① 快进状态 ② 做对事 ③ 不丢知识。

**总纲**：把规则 / 文档 / 流程当**成本**精算（它消耗最稀缺的资源——AI 的 context 注意力），不当**资产**积累。越全越放心、写了就留 = 反模式，会让治理只增不减、长成官僚部门。

**四条不变原理**（一切治理动作的判断标尺，过不了不做）：
1. **为消费者而生，无消费者即废**：从「谁用、在哪用」倒推。没消费者的规则不是中性的，是负担。
2. **腐烂是默认，解药靠「持续自动」非「周期意志」**：校验绑进每次工作流自动跑才扛得住；靠人周期维护（评审会 / 季审）是被点名淘汰的失败模式。
3. **增量进唯一真相（SoT）**：只记「在变什么」，再归档进生长的真相文档；不一次写大而全、不多处抄副本。
4. **最简默认，复杂度要「挣来」**：每条规则默认不存在，必须由真实发生的失败证明自己值得留。

**为 AI 写**：真正读这些文件的是 AI，每次对话开头自动塞进 context。AI 要的是「准、全、不打架、找得到、信号密度高」——不是「精简到人脑记得住」。文件该短，是因为 context 是有限资源、token 多了会「context rot」（回忆力下降），**短是为性能不是为凑数**。同时保持「right altitude」：宪法抽象、AGENTS 中观、runbook 具体，分层别推平。

**判据**（建每一块时自问）：业内验证过的留（Anthropic long-running agent harness / OpenAI AGENTS.md / karpathy 编码准则 / ADR / OpenSpec delta+archive / fitness functions / policy-as-code），自创的「管治理的治理」砍。

## ⚠️ 明确警告：别建这些会长成官僚的重机制

下面这些是本 playbook 母版项目**亲手砍掉的**——它们聪明、看着严谨，但全是「为治理而治理」的元层，靠从没运转的触发器维持，只增不减。**建治理时一个都别造**：

- ❌ **控制点 / 机制注册表**（「机制出生登记制」「守护守护者」的元 lint）——查「有没有没登记的规则」是最聪明也最重的元层，砍。
- ❌ **季度审查 / 定期治理审查（季审）**——纠错的命门挂在「从没开过的会」上，是被 fitness functions 点名淘汰的失败模式。用事件驱动纠错取代。
- ❌ **仪式预算 / one-in-one-out / 每轮仪式数上限**——为人脑记忆凑数，对 AI 无意义。
- ❌ **ROI 举证制度 / 治理元决策当活规则天天供**——给治理立的自我服务规矩，零兑现。元决策归档成历史 story，不当活规则。
- ❌ **L1-L4 黑话溯源链 / `[来自 L?]` 标签**——逼每条规则追溯到「真理层」的形式主义。母版连用户自己都看不懂，对北极星的第一消费者失效。宪法用大白话。
- ❌ **为「≤30 行 / ≤80 行」硬凑数**——文件该短是为 context 预算，不是为命中某个数字。别为行数较劲、别为凑数删掉有信号的内容。

> 元教训：协调器「有规则但不执行」比「没规则」更糟；自创框架的高峰也是埋病根的高峰。最好的治理是大胆删和合并，不是发明新制度。

## 你要为本项目建出这套（结构照此，内容填项目自己的）

### ① `CONSTITUTION.md` — 北极星（不变真理 · 人管 · AI 不得自动改）

**为人和 AI 都一读就懂**。大白话，不要 L1-L4 黑话，不要为凑行数硬删。失焦时回这里找方向。必须含：

- **卷首一句**：人管这份（不变的），AI 不得自动改——要改用 `constitution:` 提交 + 用户拍板。
- **SoT 根（做什么=怎么运转，同一条根）**：一句话说清本项目把「什么散乱状态」收进一个**唯一真相源（SoT）**——同一个事实只有一个权威出处，不许多份副本各说各话。这是产品。**这条根也管治理自己**：知识、规则同样一处权威、不抄副本。**做产品的方法，就是治理自己的方法。** 求真是命根：以真相源为准，不靠记忆猜，不知道就说不知道。
- **四条不变原理**（治理自身的不变层，上面「内核」那四条，用项目自己的大白话写）：为消费者而生 / 腐烂默认靠持续自动 / 增量进唯一真相 / 最简默认复杂度要挣来。
- **协作合伙人**：AI 是合伙人不是应声虫——先想清楚，该挑战就挑战、拿不准就问，别猜别附和。
- **人机分工**：人定方向和底线（不变的）；AI 管怎么实现、怎么把知识织活（会变的）。
- **干活铁律（一句话总纲）**：动手前想清楚不瞎猜；只动该动的；能简单绝不复杂；说做完了必须拿证据。（完整正文在 ④ execution-discipline。）

> 长度自然短即可（母版约 25 行），但**别为命中数字硬凑**——信号密度优先。

### ② `AGENTS.md` — 接手主入口（为 AI 写 · 最小高信号 · CodeX 和 Claude 同读）

这是被两个 AI 都自动读到的入口（CodeX 原生读 `AGENTS.md`、Claude 经 `CLAUDE.md` import 它）。落地宪法。**为 AI 写、最小高信号**，必须含这几块（顺序即建议顺序）：

- **接手先读（按顺序）**：① CONSTITUTION（北极星）→ ② 知识地图（KB index，找 X 在哪）→ ③ 状态线（单线 `PROJECT` / 多线 `ROADMAP` 当前游标——此刻在哪条线、下一步）→ ④ cold-start 上手包（协作脾气 / 踩坑 / 凭据）。
- **干活铁律（凌驾一切）**：动手前想清楚不瞎猜（做过没 / 真实状态 / 权威在哪）；只动该动的、能简单绝不复杂；说做完了必须拿证据。一句话 + 链到 ④ execution-discipline 完整正文。
- **对话中：想法落到对应文件**：改方向 / 焦点 / 优先级 → 状态线（游标 + 对应线 section）；与已定决策冲突 → 暴露冲突、等用户拍、不自动执行；新认知 / 踩坑 / 洞察 → 知识库（归口见 index）。
- **结束前（必跑，不可跳）**：① 该沉淀的沉淀（决策 / 踩坑 / 状态，一处权威别抄副本）；② CHANGELOG 追加一行 `- [YYYY-MM-DD] [type] 摘要`；③ 跑 lint 到 0 error 才算完。
- **知识往哪沉（SoT：一处权威、不抄副本）**：一张路由表——需求/任务、已定决策、领域/数据源事实、踩坑/洞察、历史事件……各自唯一去向。强调**别在多处抄正文，其它位置只放指针**。
- **治理怎么自纠（非季审）**：lint 报错当场修 · evals 答不出改文件 · 用户打脸即改 · 撞见过时规则即删——事件驱动、嵌进工作流。链到 ⑤。
- **git**：格式 `tag: 说明`；`constitution:`（宪法修订）/ `roadmap:`（主线变更）需用户审；工作线程不 merge 到 main。

> **别加 ≤80 行硬约束**。短是目标，但靠「只放高信号 + 正文外包给 runbook」实现，不靠凑数。母版约 40 行。

### ③ `CLAUDE.md` — Claude 入口（极简 · import AGENTS）

只做一件事：让 Claude 收敛到和 CodeX 同一份规则源。首行 `@AGENTS.md` 把 AGENTS 全文拉进来，正文几句话点明启动读序 + 铁律一句话，其余全指向 AGENTS——**不在这里重复规则**（凡跨工具可见的规则一律落 AGENTS，`@import` 仅 Claude 侧生效）。

### ④ cold-start 上手包 + execution-discipline（上手最后一公里）

**`docs/runbooks/cold-start-onboarding.md`** — 「地图 → 能干活」的最后一公里。**本页是装配清单（BOM）不是知识本身**：只列「接手前要知道的 N 条 + 各自权威页」，正文几乎为零、内容靠链接（复制正文必漂移）。三段：
- **§1 协作契约（user-level）**：怎么和本项目用户协作——当合伙人不当助手 / 别让用户做设计决策 / 语音按意图理解 / 沟通具体简短 / git 保持简单等。**性质 = user-level 个体偏好，不是项目规则，不进宪法溯源链**。完整偏好可能只活在某个 AI 的私有 memory（CodeX / 无记忆接手读不到）——故此处留**可读浓缩**当唯一例外，让无记忆载体也接得住。
- **§2 工程踩坑速查**：碰到再查，纯链接指向 `KNOWLEDGE_BASE/insights/` 各权威页。
- **§3 凭据 & 基础设施速查**：凭据文件位置（**本机文件、不进 git**，新 clone 需用户提供）+ 关键基础设施指针。

**`docs/runbooks/execution-discipline.md`** — 干活铁律完整正文（AGENTS 那段的展开，灵感来源 karpathy 编码准则，正文自足）：
- **动手前 — 三问，不凭记忆**：执行任何有成本操作（抓取 / 切流量 / 批量改 / 发版 / 删文件 / 发凭据）前——① 做过没？（搜 CHANGELOG / memory / 会话，别重做已做过的）② 真实状态？（查数据/代码实证，不是「我以为」）③ 权威记录在哪？（查不到先补，别让结论只活在聊天里）。三问过不了不动手。
- **动手时**：① 精准修改（只动任务相关，改动范围 = 任务范围）② 目标驱动（先定清「做完=什么样、怎么验证」）。
- **宣称完成前 — 实证**：跑测试 / 查数据 / 端到端验证，别拿残缺 grep、截断输出、旧缓存当全貌，尤其与刚做过的事矛盾时。
- **衰减兜底**：铁律多是语义自觉、机器实时拦不住，是高衰减候选——兜底见 ⑤ 治理自纠（事件驱动、非季审）。

### ⑤ `docs/runbooks/governance-self-correction.md` — 纠错机制（取代季审）

纠错不靠定期审查（季审是死触发器），绑到本来就会自动 / 高频发生的事件。必须含：

- **事件 → 动作表**：

  | 事件（自动 / 高频发生） | 纠错动作 |
  |---|---|
  | lint 报 error/warn（每次结束前 / commit 时跑） | 当场修到 0 error，别留到以后 |
  | evals 冷启动题某题答不出（大改 / 换模型时跑） | 改那道题对应的**承载文件**（修文件不修答案）——答不出 = 那块治理失效了 |
  | 用户质问 / 打脸（"这不对"、"你又附和了"） | 当场改 + 沉淀（踩坑进 insights、协作偏好进上手包） |
  | 干活时撞见规则过时 / 自相矛盾 | 当场改（SoT 一处权威只改一处）+ 记踩坑。**过时规则 = 有害，删不留**（错的指令比没指令更糟） |

- **加规则三问准入**（落地四原理，过不了不加）：① 有具体消费者吗？② 是真实发生过的失败挣来的，还是预防想象出来的？③ 机器拦得住吗——拦不住的别塞进 lint，靠 evals + 用户在环。
- **砍规则判定**：「没人用」= 没有任何文件链它（孤儿，grep 反链可判）+ 连续 ≥50 commits 无触碰、或干活时撞见即判 → 标候选、确认后砍。**用 commit 计数 / 事件触发，不挂日历周期**（日历周期就是被砍掉的季审）。过时 / 没人用的规则有害，删不留。
- **收尾提示**：治理大改后跑一遍 ⑦ 验收套件确认没弄坏哪个环节。

### ⑥ lint 精简核心检查 + pre-commit 真门禁（随 clone 生效）

**`scripts/knowledge-lint.ts`**（或等价脚本）—— 只查机器能确定性判断的，语义判断留给 evals + AI。核心检查（约 5–8 条，**别膨胀到 20**）：
- 断链：被引用的 `.md` 路径存在（`fs.existsSync` 校验）。
- 关键治理文件在不在（CONSTITUTION / AGENTS / 状态线 / 上手包）。
- 该沉淀的丢没丢：CHANGELOG 新鲜度（最新事件 >N 天报警）+ 状态线 `Last Updated` 跟得上 CHANGELOG。
- 状态线当前游标段存在（多线工作台时）。
- KB index 与实际文件一致。
- （可选）可机检的流程规则：CHANGELOG 行格式 `- [YYYY-MM-DD] [type]`、commit 带 type——这是 policy-as-code 的精神，**仅限机器能可靠判断的**，判断不了的别塞。
- 末行打印 `合计: N error / ...`，exit code 随 error 数（≥1 error 时非 0），让 pre-commit 能当门禁。

**`.githooks/pre-commit`** —— 直接调 lint 当真门禁（单一信源、不重抄阈值）：

```bash
#!/usr/bin/env bash
set -e
if command -v npx >/dev/null 2>&1; then
  echo "🔍 治理 lint（pre-commit 真门禁）..."
  npx tsx scripts/knowledge-lint.ts
else
  echo "⚠️  未找到 npx/node，跳过（CI 兜底）"
fi
```

**随 clone 生效**（关键，否则等于没装）：`package.json` 加 `"prepare": "git config core.hooksPath .githooks || true"`，`npm install` 时自动把 hooksPath 指到版本化的 `.githooks/`。各工具的 Stop hook / CI 都跑**同一个 lint 脚本**（单一信源）。

> ⚠️ 纯自觉必衰减、hook 不随 clone = 没装。执行点要 pre-commit（提交前真门禁）+ CI（push 后审计留痕）+ 每轮结束自跑——但**不要**再加「季审 cron」那一档（那是被砍的死触发器）。

### ⑦ `docs/runbooks/governance-acceptance-test.md` — 验收套件（用 AI 验 AI）

治理大改后（动宪法 / AGENTS / 砍或加规则）的端到端回归标准。**起一个无记忆 AI**（CodeX 或清空记忆的 Claude），只许读治理文件，逐用例真实应对——全过 = 治理健康；过不了 = 那块「治理没写清」，按 ⑤ 自纠「修文件不修答案」。

覆盖治理全生命周期 **6 环节**，每环节 2–3 例（母版共 18 例），每例含机器可查判据（grep / git diff / 工具轨迹）+ 语义评分要点：
1. **接手 + 理解**：无记忆 agent 按读序复述「我们做什么 / SoT 根 / 当前在哪条线 / 协作脾气 / 凭据去哪找」+ 复述四条不变原理，每条标出处文件。
2. **干活铁律**：给「已做过的事」让它先过三问别重做 / 给「带无关旧写法的修复任务」看它是否精准修改不溢出 / 给「上线后汇报」看它是否实证再宣称。
3. **对话路由**：切线 → 改游标 / 撞已定决策 → 暴露等拍 / 新踩坑 → 进 insights 单处权威。
4. **知识沉淀 + SoT**：多轮收尾分流到各自权威文件 / 识别并拒绝抄副本（同一正文进多处）/ 过时事实当场删 + 改宪法请求停下等拍板。
5. **结束前**：跑完三步（沉淀 + CHANGELOG 合规一行 + lint 0 error）/ 发现并纠正非法 CHANGELOG 格式 / 焦点转移要同步状态线游标 + Last Updated。
6. **自纠代谢**：lint 死链当场修到 0 / 撞见过时规则即删不是标注保留 / 加新规则先过三问。

**轻量子集 = evals 题库**（可单独建 `governance-evals.md`）：从环节 1–2 抽 10 来道选择/问答题，**日常 / 换模型 / 换主力工具时跑**（不排期季审），≥8/10 过；连续两题失败的领域 = 那块承载文件失效，修文件不修答案。冷启动可干活题须 CodeX + 清空记忆 Claude 各跑一遍（验证不依赖私有 memory）。

> 套件顶部留一张「历史成绩」表（日期 / 触发 / 结果），如实记录漏跑与失败——诚实是这套东西的灵魂，绝不能在重构里弄丢。

## 状态线形态（按项目并行度选）

- **单线 / 早期 → `PROJECT.md`**：当前阶段（一句话）· 当前焦点 · 指针（链向其他治理文件）。
- **多线并行（同时推进多个长期方向）→ `ROADMAP.md` 多线工作台**：顶部「当前游标」（≤3 行，此刻在哪条线 + 下一步）+ 每条线一块（状态 / 下一步 / 开放问题 / 指针）+ 折叠历史区。⚠️ 多线项目用单焦点 PROJECT 必漂（切线即过期）。**单线起步、长出多线时再把 PROJECT 并入 ROADMAP 升级。**
- 这对应 Anthropic long-running harness 的 **progress 文件**（跨无记忆 session 的状态/进展账本）。游标 = 失焦找回的活锚。瘦身：游标别写成 mini-report、别把项目史堆进折叠区。

## 建完之后（常驻运转，已写进上面 AGENTS / 各 runbook）

- **每次启动**：CLAUDE.md → CONSTITUTION → 知识地图 → 状态线（游标）→ cold-start 上手包。
- **每轮结束**：沉淀（一处权威）+ 更新状态线（游标 / Last Updated）+ CHANGELOG 追加一行 + 跑 lint 到 0 error。
- **纠错**：事件驱动（lint / evals / 用户打脸 / 撞见即改），不排季审。
- **大改后**：跑 ⑦ 验收套件（无记忆 AI 端到端），把成绩如实记进历史表。

**自证**：这套架构 ≈ Anthropic 官方长效 agent harness（progress 文件 + 失败尝试记录 + 描述性 commit + 启动先跑基础检查立即修），不是自创、有官方背书。建的时候始终拿四条原理当标尺：**服务谁？靠自动还是意志？增量进 SoT 还是又一副本？复杂度挣来了吗？** 过不了不建。
