# 判例库(品味的判例法)

> 品味无法写成抽象规则(言不尽意),只能积累判例让执行者类比插值。执行者越强,判例杠杆越大——这是唯一随模型换代**增值**的治理资产。
> 采集时机:负责人每次纠正、每个体现取舍的裁决,由执行者当场代笔落档,负责人过目。前十个判例的价值高于后一百个。
> 存放:一事一文(L3);高频判例的一行摘要可提升至宪法(占 150 行预算,自动接受稀释审查)。

判例格式(六字段,一事一文):
- **来源**:{项目 · 日期 · 触发方式(负责人当场纠正/裁决/上游反哺)}
- **场景**:
- **执行者做法**:
- **负责人要的做法**:
- **可泛化边界**:{若提炼得出;没有就空着}
- **载体**:{该教训已下沉/待下沉到哪个载体;描述性写法,不写指向个人机器的绝对路径}
