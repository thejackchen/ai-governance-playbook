# CHANGELOG

> 项目事件流水 · 按时间倒序 · 每轮结束追加一行（结束前三步之一）。
> 格式：`- [YYYY-MM-DD] [type] 摘要`——首句给结论，细节随后；一事一条。
> type 枚举（按项目增删）：`feature` / `fix` / `decision` / `governance` / `ingest` / `ops` / `docs` / `refactor` / `constitution` / `roadmap`
> lint 查：最新事件 >7 天 = 断档 error（漏沉淀，补一行再提交）；`- ` 行格式不合规 = warn。

- [2026-06-25] [governance] 项目治理初始化——按 ai-governance-playbook 建立治理骨架（CONSTITUTION/AGENTS/CLAUDE + 状态线 + lint 门禁 + 验收套件）。**把本行日期改成你项目真实的初始化日期，之后每轮追加新行。**
