# Commit 规范 — tag 表

> 格式：`tag: 中文说明`（或你项目的语言）。tag 表达「这次提交干了什么类型的事」，和 CHANGELOG 的 type 枚举对齐。

| tag | 用于 |
|---|---|
| `feat` | 新功能 |
| `fix` | 修 bug |
| `docs` | 文档 |
| `governance` | 治理文件改动（规则 / 流程 / lint） |
| `constitution` | 宪法修订——**需用户审、用户拍板** |
| `roadmap` | 主线 / 方向变更——**需用户审** |
| `ops` | 运维 / 部署 / 脚本 |
| `refactor` | 重构（不改行为） |

<!-- 按你项目实际增删 tag。constitution: / roadmap: 这两个动的是「不变的 / 主线」，必须用户拍板，AI 不自动提交。 -->
