# AI 治理方法论库 · AI Governance Playbook

> 给 AI 的**治理安装包**：把它交给任何项目的 AI，让它一看就建出一套**可懂 / 可执行 / 会自纠 / 可回归**的精简治理——不是一摞管 AI 的规章，而是一份给 AI 的好交接。
> 核心一句：**治理当成本精算，不当资产积累。借思想，别抄实现。**

## 怎么用（最重要）

把 [`setup.md`](setup.md) 交给你项目的 AI，说：

> **「读 setup.md，为本项目建立治理体系。」**

AI 会按四原理为你**强制建出**：`CONSTITUTION.md`（北极星 · 不变原理）+ `AGENTS.md`（给 AI 的接手主入口）+ `CLAUDE.md`（Claude 入口，import AGENTS）+ **工作台**（当前坐标 / 多线游标）+ 事件驱动的 lint 门禁与 evals 考试 + 验收套件。**建完你项目就有了和本库母版同构的精简治理骨架。**

**更快**：直接复制 [`templates/`](templates/) 里的现成占位骨架到你项目根、填掉 `{占位}`（见 [`templates/README.md`](templates/README.md)）——这是一套首跑 `lint` 即 0 error 的完整最小治理项目。`setup.md` 讲「每块**为什么**这么建」，`templates/` 给「现成的那一块」。

⚠️ `setup.md` 是**安装脚本**（建完使命完成）；建出来的那几个文件才是**常驻治理产物**——别搞混（这正是母版踩过、并写进反模式的「安装 vs 产物」角色混淆坑）。

## 想懂背后为什么

读 [`governance-architecture-essence.md`](governance-architecture-essence.md)——完整方法论：总纲 + 四条不变原理（每条带踩坑 why 与业内佐证）+ 核心机制 + 反模式 + 演化故事。**给人理解；AI 安装只读 `setup.md`。**


## 建完之后 · 老项目对齐

- **建完怎么验它行不行** → [SELF-CHECK.md](SELF-CHECK.md)：机器查当硬门槛、考新 AI 当体检报告（**不打分判达标**，只告诉你哪块没写清）。
- **已有项目（有 / 无 / 差的治理）怎么对齐** → [MIGRATION.md](MIGRATION.md)：**先砍后补、少而够用**，`templates/` 是取用菜单不是合规清单。

## 核心（这套东西的内核）

- **总纲 · 成本非资产**：业内最佳实践都把规则 / 文档 / 流程当**成本**精算（消耗最稀缺的注意力），不当**资产**积累。病根永远是反了——把治理当资产「越全越放心」，于是只增不减。
- **四条不变原理**（治理自身的不变层，放进宪法当判断标尺）：
  1. **为消费者而生，无消费者即废**——一切从「谁用、在哪用」倒推；没消费者的规则不是中性，是负担。
  2. **腐烂是默认，解药是「持续自动」非「周期意志」**——校验绑进每次工作流自动跑（fitness functions / policy-as-code），靠周期评审（季审 / 评审会）是被点名淘汰的失败模式。
  3. **增量进唯一真相（SoT）**——只记「在变什么」再归档进生长的真相源，不一次写大而全、不多处抄副本。
  4. **最简默认，复杂度要「挣来」**——每条规则默认不存在，必须由真实发生的失败证明自己值得留。
- **SoT 一条根**：**做产品的方法，就是治理自己的方法**——产品怎么收敛真相，治理就怎么收敛自己（同一事实一处权威、不抄副本）。
- **为 AI 写**：真正的读者是 AI——context 稀缺、有 context rot，要的是「最小高信号 token 集」+「right altitude」（别太具体也别太抽象），不是「精简到人脑记得住」。
- **事件驱动纠错**（取代季审）：纠错绑到本就高频自动发生的事件——lint 报错当场修 / evals 答不出改文件 / 用户打脸即改 / 撞见过时规则即删；不另设仪式、不排期。
- **用 AI 验 AI**：起一个无记忆 AI、只许读治理文件、逐用例真实应对——全过 = 治理健康，过不了 = 那块没写清（修文件不修答案）。这是唯一能真测「治理有没有用」「换模型扛不扛得住」的仪器。

判据：**业内验证过的留**（Anthropic long-running agent harness / OpenAI AGENTS.md / karpathy / ADR / OpenSpec / fitness functions / policy-as-code），**自创的「管治理的治理」砍**（季审 / 仪式预算 / ROI 制度 / 控制点注册表）。

## 共建

某项目踩出可迁移的新原理 / 新机制 → 加进 essence。项目特定的（具体规则 / 数值 / 业务）留本地，别污染本库。
