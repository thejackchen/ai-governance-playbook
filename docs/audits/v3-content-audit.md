# v3 内容审计

> 审计范围：`v1-pre-kit`、v2.0-v2.2 当前文件、模板与脚本。目标不是保留历史完整性，而是判断每个概念是否仍值得进入现行治理。历史原文由 Git tag 保留。

## 核心概念

| 旧内容 | 裁决 | 原因与v3落点 |
|---|---|---|
| 治理是成本，不是资产 | 保留 | 仍是防增生的总原则，进入Profile和规则死亡条件 |
| 为消费者而生 | 保留 | 升级为规则准入第一问 |
| 腐烂是默认，持续自动优于周期意志 | 修正 | 自动周期触发本身有效；应反对的是靠人记得执行，而不是反对schedule。进入触发模型 |
| 增量进入单一真相SoT | 保留并澄清 | 允许可重复生成且有漂移检查的派生副本 |
| 最简默认，复杂度要挣来 | 保留 | 改为Lite/Standard/High Assurance，不再一套全装 |
| 五块地基S1-S5 | 保留 | 作为规则分类，不作为额外文件体系 |
| L0-L3载体阶梯 | 替换 | 单轴混淆触发时点与执行强度，替换为“trigger + effect”二维描述 |
| 同一载体二犯即证伪 | 修正 | 二犯触发机制审计，不机械升级；可能是判据、覆盖或绕过问题 |
| 错误编译成防线 | 保留 | 事故回路继续使用，但允许显式接受风险 |
| 经验第二次使用才沉淀 | 修正 | 普通经验适用；安全、合规、不可逆、外部合同首次即沉淀 |
| 恒温器三标准 | 合并 | 腐烂可发现、修复便宜、不二犯并入五项有效性公式和事故回路 |
| 每周单一心跳 | 降级为Standard | 对原型过重；保留机器触发，取消“所有项目必装” |
| AI判断层 | 保留并降权 | 只读建议层、固定输出、不得成为唯一硬门禁、执行器中立 |
| 能力清单生成器 | 移出核心 | Regex无法覆盖CommonJS、微信Page等；后续作为可选extension并要求项目适配器 |
| 宪法150行/registry30条 | 改为Profile预算 | Lite 12、Standard 24、High Assurance 40；短是手段，不是目标 |

## 安装与收尾规则

| 旧规则 | 裁决 | 原因与v3落点 |
|---|---|---|
| 每个session必须写CHANGELOG | 删除绝对化 | 只读评审和讨论不制造流水；仅仓库发生有意义变化时记录 |
| 每个session必须commit | 删除 | 提交需要任务授权、验证通过和干净边界；不能替用户提交无关改动 |
| 动手前任务契约三行 | 降级 | 对复杂、模糊或高风险任务启用；简单任务不强制仪式 |
| 新原语对齐门 | 保留并改名 | 架构级/心智模型变更进入ADR或questions，不把所有新抽象都升级 |
| 结束前固定五步 | 改为change-aware | 根据改动类型选择状态、决策、事故、验证和提交动作 |
| CI空提交真实跑一次 | 保留于Standard | 能验证workflow存在；但必须配required check才称阻断 |
| pre-commit必装 | 降级为Standard | 本地可绕过，只是快速反馈，不是共享信任锚 |
| 禁止skip/mock制造假绿 | 保留 | 有实验证据且适用于所有Profile |
| 不虚报完成 | 保留 | 完成必须附可复现证据 |
| 不预防式立法 | 修正 | 合规、资金、安全与不可逆风险可由责任和合同提前建立 |

## 新增边界：目录与前端

- 仓库目录结构进入公共治理，但只治理顶层职责、所有权、依赖边界和生成物位置；不把每个文件路径写进宪法。
- 前端统一设计语言属于项目设计系统，不是通用治理正文；以可选extension提供token、组件、交互、可访问性和视觉回归的治理骨架。
- 对两者都采用同一原则：正文只有一个权威位置，能机器判断的进入lint/测试，主观设计判断保留人工评审。

## 对当前v2文案的审计

- “每session治理阅读从4-5份降到0”改为“无需额外工具读取”，因为自动注入仍消耗context。
- “20个session”“2/2”保留为早期证据，明确样本与泛化限制。
- “Codex没有Session Hook”删除；当前Codex已有SessionStart、PreToolUse、Stop等生命周期Hooks。
- “配置ANTHROPIC_API_KEY即可启用AI审计”改为provider-neutral契约，适配器决定执行器。
- “模板全部照装”改为profile选择；安装报告必须说明没装什么以及升级触发条件。

## 删除原则

被本审计删除的内容仍可通过`v1-pre-kit`和`v2.2.0` tag查阅。现行主干不保留“已废弃但仍像指令”的正文，避免新执行者误把历史方案当当前规则。
