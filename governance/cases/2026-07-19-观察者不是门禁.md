# 观察者不是门禁:验证用 exit code,不用肉眼看输出

- **来源**:playbook 自身发版过程 · 2026-07-18 实证
- **场景**:命令链里 `npm test 2>&1 | grep pass/fail` 显示了「fail 1」,但 grep 匹配到内容即 exit 0,链条继续走到 commit+push——带病发布。
- **执行者正确做法**:门禁必须消费**退出码**(`npm test && git commit`),显示类命令(grep/tail/肉眼读输出)只是观察者,观察者不阻断;发现带病提交立即补修并留档。
- **可泛化边界**:一切「验证后才允许下一步」的链条——验证器与执行器之间必须是 exit code 依赖,不是视觉依赖。CI 同理:红了必须物理阻断合并,否则是仪表盘不是闸门。
- **载体**:判例;各项目 pre-commit/CI 的既有 exit code 语义。
