# 多 worktree/多 AI 并发同仓的 git 卫生(6 次复现)

- **来源**:产品中心+AI-OS · 4 个独立 session 各自付过代价 + 同日再犯 2 次
- **场景**:多执行者并发同一仓库,commit 时把并行线程的未完成工作「顺手」带进提交(CHANGELOG/index 类高频公共文件最易中招);cd 后复合命令在错误目录执行。
- **执行者做法**:`git add -A` 图省事;脚本不锚定绝对路径。
- **负责人要的做法**:①commit 前逐文件 diff 确认改动来源;②并发期永不 `git add -A`(或 add 后 restore --staged 排除清单);③脚本一律绝对路径或 `git -C`。
- **可泛化边界**:暂存区是共享的,意图不是——一切多执行者共享工作区场景成立(6 次复现,跨项目跨模型)。
- **载体**:待下沉 pre-commit(暂存文件与声明范围核对);判例先行。
