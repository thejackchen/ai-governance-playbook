# starter/ · 可 fork 的治理脚手架模板

> 起一个新「用 AI 维护的项目」时，从这里 fork 出治理骨架，填占位、配执行点、删用不上的。
> **模板是起点，不是标准。** 它给你一个站得住的结构，但每个项目的使命 / 技术栈 / 阶段不同，治理实现必须按自己的来长。

## 里面有什么

| 文件 | 是什么 | 对应 |
|---|---|---|
| `CONSTITUTION.template.md` | 北极星空壳：使命 + 4 条不变真理 + 修订机制 | 你项目「不变的真理」 |
| `AGENTS.template.md` | 驱动引擎空壳：8 节（身份/启动/对话中/结束前/自主治理/约束/归档路由/Git） | 真理怎么落地成日常动作 |
| （本文件）`README.md` | 怎么用这些模板起步 | — |

两者的关系：**宪法是「不变的真理」，AGENTS 是「真理怎么落地」**。AGENTS 每条规则都标注溯源到宪法 L1-L4——这条溯源链是整套架构能自检的根，别拆。

## 四步起步

**1. Fork** —— 复制两个 `*.template.md` 到你项目根目录，去掉 `.template` 后缀：
```
CONSTITUTION.template.md  →  CONSTITUTION.md
AGENTS.template.md        →  AGENTS.md
```

**2. 填占位** —— 全文搜 `[待填：...]` 逐个替换；替换完**删掉所有 `<!-- 注释 -->`**（注释是填写指引，不该留在成品里）。从宪法开始填——它是根，AGENTS 的规则都要能溯源回它。

**3. 配执行点（最关键，别跳）** —— 模板只能给你「文本规则」，但**纯文本规则会衰减**。给关键规则各绑一个机器执行点，让机器当那个「消费者」：
- **断链 / 行数预算 / 宪法溯源**检查 → 一个 lint 脚本，进 pre-commit 或 CI
- **结束前必跑的收尾** → git hook（否则必被跳过）
- **季度审计 / 状态新鲜度** → cron 或 CI 定时
> 规则不带执行点，不准出生。详见上级库 `governance-architecture-essence.md` §3 执行底座、§6 机制出生登记制。

**4. 裁剪** —— 删掉你项目用不上的：单人项目可砍掉「工作线程 / 协调器」分工；没有多数据源就别留命名空间预留。**宁可结构小而真被执行，不要大而沦为装饰。**

## 还差一个入口：CLAUDE.md（或你的 AI 入口文件）

上面两个文件 AI 不会自动读。你需要一个**入口文件**（Claude 用 `CLAUDE.md`，其他 agent 用其约定的入口），把启动顺序钉死，并放一条**凌驾一切的「动手前铁律」**——这是从实战萃取的、最值得保留的可迁移骨架：

```markdown
## ⛔ 动手前铁律（凌驾一切）

执行任何**有成本的操作前**（抓取 / 切流量 / 批量改 / 发版 / 跑长任务 / 发凭据），先核实、**不凭记忆**：
1. **做过没？** 这事 / 这判断之前是否已做或已有结论 —— 搜历史记录，别重做。
2. **真实状态？** 查数据 / 代码**实证**当前真相，不是「我以为」。
3. **权威记录在哪？** 查不到就先建 / 补，别让结论只活在聊天里。
> 三问过不了，不动手。

启动顺序：CONSTITUTION.md（北极星） → AGENTS.md（工作规则全集） → [你的知识索引] → [当前坐标] → [工作记忆]
```
为什么要这条：它直接防御两个最贵的反复踩坑——「做了 ≠ 状态同步」和「散落导致重复」（见上级库 §5 定律 3、4）。它补的是「启动 → 对话 → 结束」三协议都缺的**动手前**环节。

## 三条提醒（来自这库自己的铁律）

1. **借思想，别抄实现。** 这些模板给的是可迁移的*结构*，不是要照搬的*规则*。占位里填你自己的。
2. **关键规则必须带执行点。** 纯自觉的会衰减；让 lint / hook / CI 当消费者。
3. **防过度。** 结构够用就停，转向用它、验证它。别让治理自己变成一个要伺候的工程——那正是它自己写的头号反模式。
