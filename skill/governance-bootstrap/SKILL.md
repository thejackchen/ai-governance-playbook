# 治理 Bootstrap

为本项目建立或对齐治理架构。**唯一真相源是 ai-governance-playbook 仓库——本 skill 只把你引过去，不在这里复制流程、文件清单或命令**（复制了就会和仓库漂移，违反治理自己教的 SoT「一处权威、不抄副本」）。

## 照做

1. **判断起点**：
   - 全新 / 没有治理 → 走「建立」。
   - 已有治理（可能旧、可能不好）→ 走「对齐」。

2. **现读仓库、照它做**（用 WebFetch 拉取，或让用户提供仓库内容）：
   - 建立 → 读 `https://github.com/thejackchen/ai-governance-playbook` 的 `README.md` + `setup.md`，用 `templates/` 复制填空。
   - 对齐 → 读该仓库的 `MIGRATION.md`，**先砍后补**（先砍官僚、再补「少了真接不了手」的红线）。

3. **自检**（读仓库 `SELF-CHECK.md`）：机器查 lint 必须 0 error（硬门槛）；再起一个无记忆 AI 当体检报告，看哪块没写清——**不打分判达标，只指哪补哪**。

4. **守红线**：全程对照 `setup.md` 的「⚠️ 别建这些」，别加官僚（季审 / 注册表 / 仪式预算 / 给治理打分判达标）。

## 一句话原则

凡是要起新 AI 判断的（理解、接手），一律是**体检不是考试、事件触发不是定期跑、取用菜单不是合规清单**。具体怎么做、文件叫什么、命令是什么——**去仓库现读，别凭本 skill 记忆**。
